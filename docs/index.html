What is JustType?
~ 4 min read

JustType is a Sass library that creates correct typography.

Wait. Can typography be correct?

Yes. Typography was solved centuries ago, through science, by print publishers. They created rules for proportionally scaling the size of headings, creating whitespace, placing lines on the page, and more, that, when systematically applied, yield text that is a joy to read. This is one reason why books feel nicer to read in print than on the web. Of course, a screen beaming light into your eyes is another reason, but there are tons of improvements we can bring to web typography to close the UX gap.

We need to translate the rules of print into code. As it turns out, this task is tedious and weird. For example, because text has a line-height, the same value of whitespace applied to the margin of a p is larger than when it's applied to the margin of an img. There are countless minutiae like this to comb through and fix.

And, anecdotally—I've wasted countless hours throughout my career fiddling with the margins of an h3 until it "felt" just "right". I'm tired of making these imprecise, manual tweaks constantly.

So... what's the solution?

JustType. Hear me out.

It handles all of that minutiae for you. It ships with a bunch of default values that generate good, correct typography. All you have to do is call the typography mixin. Then, use all the free time you've reclaimed to write good content and make cool things. 

@use 'just-type';

main {
  @include just-type.typography;
}

This website is styled with just the default values. To customize them, call the configure mixin: 

@use 'just-type';

main {
  /* STEP 1 (optional): Configure the module. */
  @include just-type.configure(
    $line-height: 1.4,
    $font-size: 18px,
    $font-stack: (
      'heading': ('PT Serif', serif),
      'body': ('PT Sans', sans-serif),
      'mono': ('Roboto Mono', monospace),
    ),
  );

  /* STEP 2: Use the module. */
  @include just-type.typography;
}

This is especially powerful if you have a static website or blog that generates HTML from Markdown, since the typography mixin covers every element that Markdown can generate. Configure it once, then enjoy your super readable website. As an example, this website is styled throughout with JustType, and has a blog section made with Jekyll.

Going Deeper

To create rhythm and harmony within text, the size of text elements should scale at a fixed ratio as they move up the hierarchy. In other words, if p is sized at 1rem, the smallest heading should have a font-size of 1rem * ratio, the next heading up should have a font-size of 1rem * ratio2, and so on. The same applies to whitespace. JustType has these and other typographic rules baked in, which means that no matter what base configuration you give it, it will always generate correct styles.

There's a few other features too. My favorite is responsive-breakpoint; if you configure this, JustType seamlessly changes the font-size from static to responsive at the breakpoint. For example, let's say the font-size is 16px and responsive-breakpoint is 1200px. JustType calculates how much vw is equal to 16px at a viewport size of 1200px. When the breakpoint triggers, font-size is set to this vw value. Then, as the viewport increases beyond the breakpoint, font-size scales with it, "locking in" the relative size and proportions of the text. It's hard to describe in words, so I'd encourage you to resize this window and see how the text behaves.

You can access all config values by using the getter functions defined in the config file, such as config.whitespace().

Heading sans-serif
Body Georgia, serif
Mono Andale Mono, Lucida Console, monospace
